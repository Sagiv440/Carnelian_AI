Four ways to work

One window, four modes — switched from the sidebar.

Every mode is provider-agnostic. Pick any model and it just works.

01

Chat

A normal conversation with the model, streamed token-by-token — fully local with Ollama. Renders Markdown, code blocks, and citations.

streamingoffline
02

Web Search

Flip the 🌐 toggle and Carnelian runs one live web search, injects the top results as context, and answers with inline, clickable citations.

citedper-prompt
03

Deep Research

The model plans several queries, the app reads the top pages, then it synthesizes a cited report. Optionally split planning and synthesis across models.

multi-sourcereport
04

Project

A sandboxed, tool-using agent scoped to a folder. It lists, reads, writes, edits, runs commands, and (when allowed) installs — to build or fix a real project.

agentsandboxed
The project agent

An agent that actually does the work.

Point it at a project, give it a goal. It plans, edits files, runs commands — all confined to the project directory, under approval rules you set.

🧭

A Lead that delegates

The Lead orchestrator breaks a goal into subtasks and hands each to a specialist agent — then reviews their work.

▤

Live plans & activity

Watch a plan card, a per-tool activity feed, and a collapsible card for every delegated subagent — in real time.

🔒

You stay in control

Three approval modes — confirm everything, confirm destructive, or auto-run — plus an independent software-install gate.

🔌

Extend it with MCP

Connect Model Context Protocol servers to give the agent tools for GitHub, databases, the filesystem, and more.

Everything else

Power tools, all on by choice.

Selectable personas, persistent memory, offline voice, and a hardware-aware model picker — woven into every mode.

Agents

Five built-in personas (incl. the Lead orchestrator) with skills & tool allow-lists — or roll your own, per-project.

Memory

Persistent global & per-project facts, stored as portable Markdown and recalled across sessions.

Thinking

A per-prompt reasoning toggle with an effort slider — chain-of-thought shown in a collapsible block.

Voice

Read replies aloud with offline Piper TTS — one-click install, language-matched voices, zero cloud.

MCP servers

Plug in Model Context Protocol tools, resources & prompts — global or saved with the project.

LLM Browser

A hardware-aware recommender — scans your CPU/RAM/GPU and ranks Ollama models that fit, with inline download.

Attachments

Drop in images for vision models, or PDFs / DOCX / ODT / code — text is extracted into the prompt.

Budget tracking

Per-provider cost estimates and budgets for cloud usage — set a cap or mark a subscription.

Private by default

Local models never leave your machine. Cloud is off until you hand over a key.

100%

On-device option

With Ollama selected, prompts, files, and replies stay entirely on your computer.

0

Telemetry

No analytics, no phone-home. The only network calls are searches and page fetches you trigger.

.md

Portable data

Agents, memory, and chats are plain Markdown & JSON in your own folders — yours to read, edit, or delete.

Get Carnelian.

Self-contained builds for Windows & Linux — no .NET install needed on the target. Linux also ships as a .deb and a Flatpak. Or build it from source in a couple of commands.

⊞ Windows (.zip) 🐧 Linux (.zip) 📦 Linux (.deb) 📦 Linux (Flatpak)
bash — build from source
# clone & run — needs the .NET 9 SDK + Ollama
$ git clone https://github.com/Sagiv440/Carnelian_AI
$ ollama serve && ollama pull llama3.1
$ dotnet run --project src/AI_Interface
